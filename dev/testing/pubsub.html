Publishing and subscribing to events

Publish process is used to broadcast an event to independent sequences.
The publish step does not have any knowledge of his subscribers.

Procedure

  1. Create a new Sequence 'PublishTest' containing a 'Publish' step in position 1.
  2. Click Configure on the step then Manage Events.
  3. Create a new Event type called 'Test Event' and choose this on the Publish step configuration.
  4. Create a new Sequence 'SubscribeTest' containing a 'Subscribe' step in position 1.
  5. Click Configure on the step then choose 'Test Event' from earlier.
  6. Append a 'Debug' step to the sequence in Position 2. The 'Terminate' should be at the end.
  7. Select the 'PublishTest' sequence.
  8. Click Enqueue Test Event.
  9. Wait for the monitor to show all processing has finished.
  10. Click the messages tab and sort by created date showing newest messages at top.

Outcomes

  • Confirm that five (5) messages were created with the following routes
    • PublishTest#1
    • PublishTest#2
    • SubscribeTest#1
    • SubscribeTest#2
    • SubscribeTest#3
