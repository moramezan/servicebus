Killing unprocessed messages

Problematic messages should not annoy the user forever, they can 'kill' them to move them out of view.
However we don't allow users to delete messages lightly, as this would affect auditing and replay.

Procedure

  1. Create a new Sequence 'KillTest' containing an unconfigured 'Branch' step in position 1.
  2. Click Enqueue Test Event to process a message, which should fail.
  3. Wait for the monitor to graph the 'Exception' message for 'Branch'.
  4. From the Messages tab, go to the 'Exception' listview.
  5. Select the checkbox next to the message and click Kill.

Outcomes

  1. No messages should be visible in the 'Exception' listview.
  2. Exactly one message should be visible in the 'Killed' listview.
  3. The message status should be 'Completed' with some stack trace in 'Cause' field and some error in 'Exception' field.
