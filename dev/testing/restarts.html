Restarts multiple unsuccessful messages

This allows users to asynchronously re-attempt the processing of unsuccessful messages.
Usually the user might first fix the event parameters or fix the process logic.

Procedure

  1. Create a new Sequence 'RestartsTest' containing an unconfigured 'Jump' step in position 1.
  2. Click Enqueue Test Event to process a message, which should fail.
  3. Wait for the monitor to graph the 'Exception' message for 'Jump'.
  4. Create a new Sequence 'TerminateTest' with just the default 'Terminate' step.
  5. View the 'RestartsTest' sequence and configure the 'Jump' step to point to 'TerminateTest'
  6. From the Messages tab, go to the 'Exception' listview.
  7. Select the checkbox next to the message and click Restart.

Outcomes

  1. No messages should be graphed in the monitor.
