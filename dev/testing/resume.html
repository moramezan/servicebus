Resuming the message queue

Resume will begin processing new messages again, including any paused ones.
This is used to start the broker after system modifications are made.

Procedure

  1. From any Messages listview, click Pause Queue.
  2. Create a new Sequence containing a 'Debug' step in position 1.
  3. Click Enqueue Test Event, which will exist in a paused state.
  4. From any Messages listview, click Resume Queue
  5. Wait for the monitor to show all processing has finished.

Outcomes

  1. No messages should be graphed in the monitor.
  2. Broker should run and send exactly one Debug email.
