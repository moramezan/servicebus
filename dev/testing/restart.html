Restart one unsuccessful message

This allows users to synchronously re-attempt the processing of unsuccessful messages.
Usually the user might first fix the event parameters or fix the process logic.

Procedure

  1. Create a new Sequence 'RestartTest' containing an unconfigured 'Jump' step in position 1.
  2. Click Enqueue Test Event to process a message, which should fail.
  3. Wait for the monitor to graph the 'Exception' message for 'Jump'.
  4. From the Messages tab, go to the 'Exception' listview.
  5. Open the sole message and click Restart, which should fail leaving exactly the same stack trace etc.
  6. Create a new Sequence 'TerminateTest' with just the default 'Terminate' step.
  7. View the 'RestartTest' sequence and configure the 'Jump' step to point to 'TerminateTest'
  8. Again from the Messages tab, go to the 'Exception' listview.
  9. Open the same message and click Restart which should redirect back to the list view.

Outcomes

  1. Broker should not be running. The red twirly should be absent.
  2. No messages should be visible in the 'Exception' listview.
