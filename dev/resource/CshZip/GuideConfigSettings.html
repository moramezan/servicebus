Step Config and Process Settings

Configuring process behaviour

Instead of replicating a large portions of custom logic in a second or third plugin process, a process developer may instead choose to expose variation in execution logic through the use of plugin attributes.

Plugin attributes can apply to:

  • the process class as a whole via a global process custom setting, or
  • process logic at a specific step by making use of a step config object.

Process Settings (settings specific to each process)

The snippet below illustrates how to make use of class wide custom settings for your plugin. 

global class Meta {
    //..
    global SObject CustomSetting = YourCustomSettings__c.SObjectType.newSObject();
    //..
}


global void invoke() {
    //..
    //main code block
    YourCustomSettings__c settings = YourCustomSettings__c.getInstance();
    String yourField = settings.YourCustomField__c;
    //..
}

Notes

003 if you wish to surface a link to the Custom Setting of your process on the Processes list view, simply assign an instance of your custom setting SObject to the CustomSetting parameter of the plugin's ESB inner class.
011 - 012 to reference generic plugin configuration settings inside the plugin execution, simply call getInstance() on your custom setting object and access the relevent fields.

Step Config (config specific to each step)

The snippet below illustrates how to make use of a step specific configuration for your plugin. 

//..
global YourConfiguration__c StepConfig;
//..

global class Meta {
    //..
    global SObject ConfigObject = YourConfiguration__c.SObjectType.newSObject();
    //..
}


global void invoke() {
    //..
    //main code block
    YourConfiguration__c configuration = this.StepConfig;
    //..
}

Notes

005 - 009 if you wish to surface a link to step specific settings of your plugin, simply assign an instance of your step config SObject to the StepConfig parameter of the plugin's ESB inner class.
012 - 017, 002 To access step specific configuration from code inside the plugin execution, simply declare a class instance global YourConfiguration__c with the name StepConfig.

Note The ESB framework will correctly hydrate the instance for you.
