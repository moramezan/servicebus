Marker Class

The ESB Process Marker Class is an inner class template that allows any Apex Class to become an ESB Process. It describes metadata around a process including the Tag, Name, Description, Icon, Limits etc. This inner class must be present with a Description for an Apex class to be discoverable as a Process.

Usage

The following custom process uses examples of all of the available Marker Class properties: 

//this is a Custom Process Class
global with sharing class ChatterPlugin {
    
    //this is a inner Marker Class
    global class ESB {
        global String Description = 'This plugin creates a Chatter FeedItem Post on the user.';
        global String Tag = 'Acme';
        global String Name = 'Acme Chatter Process';
        global String Icon = 'comment';
        global String HelpUrl = Page.ChatterPluginHelp.getUrl();
        
        global Map<String,Object> Inputs = new Map<String,Object>{
            'acme__ChatterBody' => 'Text to use for the body text of the post.',
            'acme__ChatterLinkUrl' => 'Link that will be shared as the subject of the post.'
        };
        
        global Map<String,Object> Inputs = new Map<String,Object>{
            'acme__FeedItemId' => 'Id of the inserted FeedItem.'
        };
        
        global SObject ProcessSetting = ChatterSetting__c.SObjectType.newSObject();
        global SObject StepConfig = ChatterConfig__c.SObjectType.newSObject();
    }
    
    //process logic omitted for clarity
    override global String toString() {
        return '[{}]';
    }
    
}
