Introducing Messages

Messages represent real-world events. These occur all the time, for example, a clock strikes midnight, or a bank account becomes overdrawn.

A message wraps attributes supporting an event in a common structure.

A process will perform work upon receipt of a message.

All messages enter a queue, which can be seen from the Messages tab.

List View buttons

Pause Queue halts the processing of new Messages. New Messages will be queued up outside their destined Sequence. Old Messages already in a Sequence will filter through to completion.

Resume Queue resumes the processing of all Messages.

Detail Page buttons

Process Message forces this Message to be processed immediately. You can use this button to process individual Messages when the system is paused, or to reprocess any Message that could not be processed.

Show Data takes you to the native view of the underlying data pointed at by the Message. This button can be used to edit or audit the data of any specific Message handed into a given Process.
