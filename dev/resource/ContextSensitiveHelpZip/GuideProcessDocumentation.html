Documenting event parameters

It is important to advertise both:

  • the parameters your process expects to be available on the input event and
  • the parameters your process either adds to or modifies on the output events collection.

The mechanism for achieving this is described below.

Input event parameters

The snippet below illustrates how to document parameters that your process expects on any input message. 

global Map<String,String> Inputs = new Map<String,String>{
    'c:Url' => 'STRING representing url to scrape.'
};

Notes

001- 003 succinctly describe each parameter that the process depends on

Output event parameters

The snippet below illustrates how to document parameters that your process either adds to your output message collection or modifies on your output message collection. 

global Map<String,String> Outputs = new Map<String,String>{
    'RecordId' =>'ID referencing a document containing the contents of the scraped input url'
}

Notes

001- 003 succinctly describe each parameter that the process either adds or modifies.
