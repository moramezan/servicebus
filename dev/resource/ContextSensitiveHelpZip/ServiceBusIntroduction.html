Introducing Services framework

Services framework is the only high performance messaging platform native to Salesforce. It allows systems to react to real world events. You can model sagas, asynchronous services, and recover from errors without data loss using the drag-and-drop sequence builder.

Services framework is predicated on the notion that work is broken up into services that respond to events.

Using the Services framework encourages:

  • reactive, extensible and event-driven models,
  • testable components which have no knowledge of each other, and
  • coding to an interface that responds to events.

Services

A Service is a single piece of "encapsulated goodness" with inputs and outputs. Business logic should be divided into distinct services exhibiting the following implementation characteristics:

  • a single Apex class per service,
  • a single input event injected into each service, and
  • the emission of zero, one, or many events as a collection from each service.

After a service receives and acts on an event, it's output events are routed to a subsequent service. Services can split, combine, or alter data, and call Salesforce APIs.

Events

In the real world events occur all the time. For example, the clock strikes 12PM, the fuel tank reaches empty, etc. To represent real world events in computer systems, we create a data structure called an Event which contains relevant details about the real world event. The primary purpose of an event is to provide a useful mechanism to communicate between services.

Messages

Events are persisted in Salesforce via a SObject called a Message. The terms Event and Message are therefore used interchangeably but represent the same concept for the purposes of ths guide.

Sequences

A Sequence is an arrangement of multiple processing tasks called Steps. Each Sequence is known by a friendly name, for example 'ProcessInvoices', which should in some way indicate the nature of the work that it's processing steps will perform.

The Services framework will advance Events through the Steps of a Sequence in the order you specify using a drag and drop mechanism. A typical sequence will contain many steps.
