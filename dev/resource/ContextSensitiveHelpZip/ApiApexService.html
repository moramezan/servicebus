Apex Services

A service is a single, logical program with inputs and outputs in the form of events. Services are implemented as Apex classes. Each service handles one event, and may output zero, one, or many new events. All the traits available to service authors can be availed using a Service Definition.

This example shows the use of the service definition class, which you will see throughout tutorials and documentation: 

global class ExampleService extends Abstract.Service {
    
    global class Definition extends Abstract.Service.Definition {
        String Description = 'Example Service';
    }
    
    override public void callouts(Map<String,Object> event) {
        //perform service callouts
    }
    
    override public List<Map<String,Object>> execute(Map<String,Object> inputEvent) {
        //implement service logic
    }
    
}
Note All output event parameters must be JSON serializable, to ensure they can be persisted between execution contexts.

The service bus writes state to the database before and after service execution. This allows you to recovery from errors.
