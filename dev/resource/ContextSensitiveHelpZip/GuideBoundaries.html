Package Boundaries

External Boundary - Inbound

External events (for example "Time is 12:01" or "Stock price changed") are not directly consumable by the Services Package. Instead, Service configurators make use of the Services.Enqueue.invoke() method to transform an external events into event messages that are consumable by the Services Package.

The Services.Enqueue.invoke() method supports several enqueue approaches, namely:

  • Map<String,Object> event = new Map<String,Object>{ 'ToSequence' => 'MySequence'};
  • Map<String,Object> event = new Map<String,Object>{ 'ToService' => 'MyService'};
  • Map<String,Object> event = new Map<String,Object>{ 'ToConfig' => 'MyConfig'};

Example Usage:

Map<String,Object> event = new Map<String,Object>{
   'ToSequence' => 'MySequence',
   'RecordId' => '001000000000000aaa'
};
Id jobId = services.enqueue.invoke(event);
ToSequence
defines which sequence will respond to the injected event,in this case MySequence
RecordId
defines the SObject that will be subject to the processing steps, in this case Account instance

Use the Services.Enqueue.invoke() method whenever the data related to the external event is retrievable as a Salesforce object using the object's Id.

(see Api Guide for further details.)

External Boundary - Outbound

Any service is free to communicate outside the boundary via:

  • HTTP callouts
  • the invocation of a workflow rules
  • DML operations, and/or
  • the sending of one or more email messages

Internal Boundary

All Sequences have a Terminate service as their final processing step. Conceptually, this is very similar to a 'buffer stop' at the end of a railway line.

A Terminate processing step is automatically inserted when the sequence is created.

Sequences may interact with one another via one of four standard pre-packaged services, namely:

  • Jump
  • Transaction
  • Publish
  • Subscribe
