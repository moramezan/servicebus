Framework Boundaries

External Boundary - Inbound

External events (for example "Time is 12:01" or "Stock price changed") are not directly consumable by the ESB framework. Instead, ESB configurators make use of the esb.Job object to transform an external events into event messages that are consumable by the ESB framework.

The esb.Job object supports several signatures, namely:

  • esb.Job(String sequenceName, Id recordId)

Usage:

sequenceName
defines which sequence will respond to the injected message event.
recordId
defines the SObject that will be subject to the processing steps

Use the esb.Job(String sequenceName, Id recordId) constructor whenever the data related to the external event is retrievable as a Salesforce object using the object's Id.

External Boundary - Outbound

Any process is free to communicate outside the framework boundary via:

  • HTTP callouts
  • the invocation of a workflow rules
  • DML operations, and/or
  • the sending of one or more email messages

Internal Boundary

All Sequences have a Terminate process as their final processing step. Conceptually, this is very similar to a 'buffer stop' at the end of a railway line.

A Terminate processing step is automatically inserted when the sequence is created.

Sequences may interact with one another via one of four standard pre-packaged processes, namely:

  • Jump
  • Branch
  • Soubroutine
  • Publish
  • Subscribe

(see Core processes Guide for further details.)
