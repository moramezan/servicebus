Introducing Processes

A Process is a single, logical program with inputs and outputs in the form of Messages. Logic that splits data, combines data or alters data should be encapsulated in distinct processes. When a process receives and acts on a message, it may output multiple new messages that go to subsequent processes.

Processes are implemented as Apex classes. The Enterprise Service Bus framework ("ESB") contains by default a number of ready-made processes, including:

  • Schedule
  • Publish and Subscribe
  • Branch, Jump and Subsequence
  • Audit and Wiretap
  • Mark and Gate

See the tutorial: How to discover ESB Processes.

List View Buttons

Discover Processes can be used to find new Processes. Click this button after authoring or installing Processes to make them available.

Detail Page Buttons

View Apex Class will show you the Apex Class that contains the underlying logic for any given Process.

View Settings takes you to the available Settings for a Process. (Note: Not all process have Settings)

View Documentation shows the supporting documentation provided by the author of a given Process.

Discovering Processes

Before a process written in Apex can be referenced as a step in a sequence, it must first be discovered and registered in the Enterprise Service Bus framework.

This discovery job will be launched automatically when installing the package or trying to build a sequence for the first time.

There is also a mechanism for discovering processes manually and is initiated by clicking the [Discover] button from the Processes list view.
