Sequences

What is a sequence?

A Sequence is an arrangement of processing tasks called steps. Each Sequence is known by a friendly name, for example 'HandleRefund', which indicates the nature of the work that it's steps will perform.

Specify the order of the different steps using drag and drop. The Service Bus advances your events through the steps of the sequence. A typical sequence contains many steps.

The last step of every sequence is a 'Terminate' service. This step swallows events that reach the end of the sequence.

How do I enqueue an Event into a Sequence?

event = {'ToSequence' => 'MySequenceName'};
job = new ServiceBus.Job(event);
System.enqueueJob(job);

How do I enqueue an Event into a Service?

event = {'ToService' => 'MyServiceName'};
job = new ServiceBus.Job(event);
System.enqueueJob(job);

When would I enqueue to a Sequence instead of a Service?

  • Whenever your business logic consists of more than one service.
  • Whenever you need to split and then aggregate multiple events.
  • Whenever there is a requirement for services to communicate with each other.
