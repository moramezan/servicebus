Debug Service

The Debug sends you an email containing all the event parameters that pass through by it. It is primarily a learning and debugging tool.

Note Debug is a pass-through service. It does not alter the event parameters or have any other side effects.

Debug Service Config

Normally the email sent to the Services Integration User. Or you may nominate a specific user to receive these emails.

To change the Debug recipient:

  1. Open the Services tab,
  2. Scroll down to Debug, then click Config. This will take you to the Debug Service Config view.
  3. You can modify the organization-wide setting. Set the User Id who should receive the emails.
