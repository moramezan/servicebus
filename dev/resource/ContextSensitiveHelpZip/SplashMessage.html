Messages

What is a message?

A message is a container for holding a real world event. These events occur all the time, for example a clock strikes midnight, or a payment is requested. Wrapping each event in a message gives a handle to identify it.

  1. The event JSON can be seen inside each message.
  2. If a service ever fails, the message remains visible for retry.

How do I deal with failed messages?

All messages are stored in a queue for review. You can either:

Restart
to reprocess it again synchronously, after fixing the apex code or the event;
this is immediate and highly responsive, used to the benefit of developers.
Rebuffer
to requeue it again asynchronously, after fixing the apex code or the event;
this is asynchronous, and can be performed on large groups of messages.
Kill
to dismiss it, if the failure was expected and acceptable;
this should only be used when the situation is unrecoverable.
