Step Config and Service Settings

Configuring service behaviour

Instead of replicating a large portions of custom logic in a second or third service, a service developer may instead choose to expose variation in execution logic through the use of service attributes.

Service attributes can apply to:

  • the service class as a whole via a global service custom setting, or
  • service logic at a specific step by making use of a step config object.

Service Settings (settings specific to each service)

The snippet below illustrates how to make use of class wide custom settings for your service. 

global class Summary {
    //..
    global SObject ServiceSetting = YourCustomSettings__c.SObjectType.newSObject();
    //..
}


override global String toString() {
    //..
    //main code block
    YourCustomSettings__c settings = YourCustomSettings__c.getInstance();
    String yourField = settings.YourCustomField__c;
    //..
}

Notes

003 if you wish to surface a link to the Custom Setting of your service on the Services list view, simply assign an instance of your custom setting SObject to the CustomSetting parameter of the service's Summary Marker Class.
011 - 012 to reference generic service configuration settings inside the service execution, simply call getInstance() on your custom setting object and access the relevent fields.

Step Config (config specific to each step)

The snippet below illustrates how to make use of a step specific configuration for your service. 

//..
global YourConfiguration__c StepConfig;
//..

global class Summary {
    //..
    global SObject ConfigObject = YourConfiguration__c.SObjectType.newSObject();
    //..
}


override global String toString() {
    //..
    //main code block
    YourConfiguration__c configuration = this.StepConfig;
    //..
}

Notes

005 - 009 if you wish to surface a link to step specific settings of your service, simply assign an instance of your step config SObject to the StepConfig parameter of the service's Summary Marker Class.
012 - 017, 002 To access step specific configuration from code inside the service execution, simply declare a class instance global YourConfiguration__c with the name StepConfig.

Note The Service Bus will correctly hydrate the instance for you.
