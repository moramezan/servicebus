Branch, Jump and Subsequence services

Execution logic is often broken into two or more sequences. A need therefore arises to connect the business logic contained in these sequences

The Branch, Jump and Subsequence services allow messages to transition between sequences.

  • The Jump service directs execution flow to the first step in an alternative sequence.

    Note Steps in the current sequence following a Jump step will not be executed.

  • The Branch service directs execution flow to the first step in an alternative sequence AND allows the remaining steps in the current sequence to complete.

    Note Unlike the Jump service, steps in the current sequence following a Branch step will still be executed

  • The Subsequence service directs execution flow to the first step in an alternative sequence.

    Note When execution reaches the Terminate step in the alternative sequence, execution flow returns to the step immediately following the Subsequence step of the current Sequence

Config

To specify the sequence to jump to, click Configure next to a Jump sequence step. Clicking the configure link will redirect to the Jump config view. In the Sequence field, enter the name of the sequence to jump to.

To specify the sequence to branch to, click Configure link next to a Branch sequence step. Clicking the configure link will redirect to the Branch config view. In the Sequence field, enter the name of the sequence to branch to.

To specify the sequence to execute before returning, click Configure clink next to a Subsequence sequence step. Clicking the configure link will redirect to the Subsequence config view. In the Sequence field, enter the name of the sequence to execute.
