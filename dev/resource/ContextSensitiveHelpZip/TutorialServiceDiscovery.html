Service Discovery Tutorial

Objective

This tutorial demonstrates the discovery of new business logic. We will create a new service conforming to a standard definition so that it can be referenced by the Services Package.

Expected duration: 10 minutes.


Background Context

Before a Service can be referenced as a step in a sequence, it must first be discovered by the Services Package. It detects all Apex Classes in the organization that conform to a service definition. All valid services are made available for potential selection by the system orchestrator when building sequences under the tab.


Instructions

  1. Open the developer console. Create a new Apex class and name it MyService. Paste in the following code and press Save. 
                global class MyService extends Abstract.Service {

                global class Definition extends Abstract.Service.Definition {
                    String Label = 'My Service';
                    String Description = 'My description.';
                }

                override public List<Map<String,Object>> execute(Map<String,Object> inputEvent) {
                    return new List<Map<String,Object>>();
                }

            }
  2. Open the App menu on the top right of Salesforce and choose Services.
  3. Navigate to the Services tab. Progress is shown at the top of the page.
  4. Once discovery is complete, you will be taken back to the list view.
